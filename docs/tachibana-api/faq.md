# 立花証券 e支店 API FAQ

出典: https://www.e-shiten.jp/QA/answer14.html

## Q1: APIの動作を大まかに教えてください

クライアントがテキストを作成し、HTTPでAPIサーバーに送信して文字列応答を取得する。
ブラウザで試したい場合は、マニュアル「ブラウザからの利用方法」の「基本」シート参照。

## Q2: APIに設定する項目がわかりません

マニュアル「REQUEST I/F、注文入力機能引数項目仕様」(p5以降)に具体的なパラメータ例あり。
Python サンプルプログラムのコメントにも該当箇所が記載。

## Q3: 仮想URLの有効期限

sUrlRequest と sUrlEvent は以下で無効化される:
- ログアウト
- 同じユーザーIDでの再ログイン
- API用サーバー閉局 (03:30)

## Q4: 時刻チェック

`p_sd_date` でクライアント時刻をチェック。
形式: `YYYY.MM.DD-HH:MM:SS.TTT`（秒の小数点以下3桁）
サーバー時刻より30秒以上遅い場合はエラー応答 (`p_errno=8`)。

## Q5: sJsonOfmt による返信形式

| 値 | 形式 |
|----|------|
| "0", "2" | 項目番号、改行なし |
| "1", "3" | 項目番号、改行あり |
| "4" | 項目名、改行なし |
| "5" | 項目名、改行あり |

## Q6: p_no の設定方法

ログイン時の指定値で初期化し、その後は毎回 +1 以上インクリメントして送信。

## Q7: パスワード変更後ログインできない

記号は URL エンコードが必要:
- `#` → `%23`
- `+` → `%2B`
- `/` → `%2F`
- `:` → `%3A`
- `=` → `%3D`

## Q8: 240回でAPIが反応しなくなる

プロバイダの技術仕様による制約。
別プロバイダに変更するか、アクセス頻度を下げる。

## Q9: 返済注文取消後の再発注エラー

取消応答後すぐに再発注すると「信用建玉明細にデータなし」エラー。
`CLMShinyouTategyokuList` で返済可能数量を確認してから再発注。
連続した注文・訂正・取消時は口座状況確認後に後続電文を送信。

## Q10: 歩み値の取得

取得不可。

## Q11: APIが動かない

返信の `p_errno` と `p_err` を確認。
マニュアル「共通項目、認証機能」の `p_errno` 一覧で対応方法を確認。

## Q12: 可能額頻繁アクセスで高負荷注意 ★重要

**ポーリング利用は想定していません。**
EVENT I/F（サーバープッシュ）で約定通知を受け、
それをトリガーに口座状況を確認してください。
